Aaron's Sudoku Solver

Naive Start from top left corner, and find the next open cell, take a guess, and go on. If there is a conflict, backtrack.
Semi-Smarter Same as naive except that to start off, fill in cells with only 1 guess.
Smarter Instead of looking for first open cell, look for the cell with the smallest number of guesses, and start from there. Then go through all the other cells the same way and backtrack if needed.
Notes I spent a lot of time trying to add a lot of different heuristics such as removing guesses from neighbors, filling in forced cells after using the cell with the smallest guesses, etc. I ended up not being able to get it all to work together. Also, in hindsight I should have used a dictionary to store the board. Due to me doing this all recursively and having a 2d array as a board, there are a lot of complications and it is now too late to redo.
